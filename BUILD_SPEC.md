## 1. 목표와 범위

호주판(edm-au-ai)의 검증된 골격 — 히어로 훅 → 트랙 선택 → 단계별 진단 → 프로필 요약 → 추천 전공 → 경로 비교 → 추천 학교 카드 → 신뢰 요소 → 상담 CTA — 을 그대로 두고, 내용 축을 "성적 극복 → 명문대"에서 **"전공 선택 → 취업허가(PGWP)·이민"**으로 바꾼다.

- 범위 내: 텍스트·데이터·진단 질문·추천 로직·결과 화면 내용·배지 표시
- 범위 외: 디자인/CSS/레이아웃 변경, 백엔드 추가, 학비 실데이터 입력(후속), 이미지 신규 제작(후속)

## 2. 리포 준비

1. `edm-au-ai` 리포를 새 리포로 복제한다. 권장 이름: `edm-ca-college-ai` (종로 브랜드로 확정되면 `chongro-ca-college-ai` 등으로 변경 가능 — 이름만 다르고 절차 동일).
2. 이 킷의 `CLAUDE.md`, `BUILD_SPEC.md`, `data/`, `docs/`를 리포 루트에 복사한다.
3. GitHub Pages 설정은 호주판과 동일하게 유지한다.

## 3. 데이터 계약 (data/canada_colleges_data.json)

단일 데이터 소스. 하드코딩된 호주 대학 배열을 이 JSON 로드로 대체한다(정적 호스팅이므로 `fetch('./data/canada_colleges_data.json')` 또는 빌드 시 JS 객체로 인라인 — 기존 코드 방식을 따른다).

- `regions[]`: 5개 권역. 각 권역에 `colleges[]`(name, city, type, market_tier, strength?, tuition_intl), `tags[]`(권역 특성 배지), `adjacent_universities[]`(BC만), `excluded[]`(불어권 기관 등)
- `major_categories[]`: 11개 전공 카테고리. `pgwp_non_degree` 값이 배지·경고 분기의 유일한 기준
  - `eligible` → 초록 배지 "PGWP 가능"
  - `conditional` → 주황 배지 "조건부 — 프로그램별 확인 필요" + `caution` 문구 노출
  - `not_eligible` / `mostly_not_eligible` → 빨강 배지 "PGWP 불가(비학위 기준)" + `alternative` 문구 노출
- `policy_snapshot_2026`: 정책 고지 섹션과 결과 화면 각주에 사용
- `tuition_intl`이 null이면 학비 칸에 "상담 시 안내" 출력. **숫자 임의 기재 금지**
- `market_tier`: 추천 우선순위 core > major > secondary. `special`은 특화 매칭(예: Olds=농업)에서만 노출

## 5. 진단 플로우 명세

기존 호주판의 스텝 UI·상태 관리 방식을 그대로 재사용한다. 질문·선택지만 아래로 교체.

### 5.1 트랙 A — 커리어·이민 트랙 (6문항)

| # | 질문 | 선택지 → 내부 값 |
|---|---|---|
| A1 | 현재 상태를 알려주세요 | 고교 졸업(예정) `hs` / 대학 재학·휴학 `uni_current` / 대학 졸업 `uni_grad` / 직장인·경력자 `worker` |
| A2 | 가장 중요한 목표는? | 캐나다 취업·영주권 `settle` / 해외 경력 쌓고 귀국 `experience` / 아직 탐색 중 `explore` |
| A3 | 관심 분야는? | major_categories 11개 + "잘 모르겠어요" `unsure` |
| A4 | 어떤 환경을 선호하세요? | 대도시 인프라 `metro` / 이민 유리·생활비 절약 `settle_friendly` / 상관없음 `any` |
| A5 | 연간 예산(학비+생활비)은? | 3천만 원 이하 `low` / 3~5천만 원 `mid` / 5천만 원 이상 `high` |
| A6 | 공인 영어 성적은? | IELTS 6.0 이상 `ok` / 5.5 수준 `mid` / 아직 없음 `none` |

### 5.2 트랙 B — 대학 편입 트랙 (4문항)

| # | 질문 | 선택지 |
|---|---|---|
| B1 | 현재 상태 | 고3·고졸 / 대학 재학 / 대학 졸업 |
| B2 | 목표 대학 지역 | BC(UBC·SFU 등) / 온타리오(토론토대 등) / 미정 |
| B3 | 희망 전공 계열 | 상경 / 이공 / 인문사회 / 예술 / 미정 |
| B4 | 연간 예산 | 트랙 A와 동일 3구간 |

## 6. 추천 로직 (의사코드)

```
// 트랙 A
1) 전공 결정
   if A3 == unsure:
       if A2 == settle → 추천 풀 = [health, education, trades] 중 A1 가중
         (uni_grad·worker → health/IT 포스트그래듀에이트, hs → trades/health 디플로마)
       else → 추천 풀 = [it, engineering_tech, health]
   else: 선택 카테고리 사용
2) PGWP 분기 (major_categories[선택].pgwp_non_degree)
   eligible     → 정상 추천 + 초록 배지
   conditional  → 추천 + 주황 배지 + caution 문구
   not_eligible → 경고 카드 출력:
       "이 전공의 컬리지 디플로마는 2026년 기준 졸업 후 취업허가(PGWP) 대상이 아닙니다."
       + 대안 3버튼: ① 인접 적격 전공 보기(business→it, hospitality→culinary,
         media_arts→engineering_tech) ② 학위·편입 트랙으로 전환 ③ 전문가 상담
3) 권역 결정 (점수제, 동점 시 앞선 순서)
   기본점수: ontario 2, bc 2, alberta 1, prairies 1, atlantic 1
   A4 metro → ontario +2, bc +2 / settle_friendly → prairies +2, atlantic +2, alberta +1
   A5 low → atlantic +2, prairies +2, ontario −1, bc −2 / high → ontario +1, bc +1
   전공 보정: trades·culinary → alberta +2 / it → ontario +1, bc +1 /
             health·education → 전 권역 무보정 / agriculture → alberta +2(Olds)
   상위 1~2개 권역 선택
4) 컬리지 선정: 선택 권역의 colleges에서 market_tier(core→major) 순으로 2~3곳,
   strength가 전공과 일치하면 최우선(예: culinary → Niagara, VCC, Holland)
5) A6 == none 또는 mid → 결과에 "어학연수·패스웨이 병행" 한 줄 추가

// 트랙 B
   B2 BC 또는 미정 → Langara, Douglas, Camosun (+ "BC 트랜스퍼 시스템" 설명)
   B2 온타리오 → Seneca, Humber, Sheridan (학위·편입 협정 중심)
   B4 low → atlantic·prairies 컬리지 1곳을 "학비 절약 대안"으로 병기
   결과에 항상: "편입·학위 과정은 PGWP 전공 제한을 받지 않습니다" 표기
```

## 7. 결과 화면 구성 (호주판 결과 레이아웃 재사용)

1. Pathway 이름·설명 (예: "보건의료 커리어·이민 경로")
2. Profile Summary: 상태 / 목표 / 관심 분야 / 선호 지역 / 예산 / 영어
3. 추천 전공 카드: 카테고리명 + PGWP 배지 + 대표 프로그램 3개(examples에서) + caution/alternative(해당 시)
4. 경로 다이어그램(텍스트 스텝): 입학 → 디플로마 2년 → PGWP 최대 3년 → 경력 12개월 → EE/PNP 검토
5. 추천 컬리지 카드 2~3곳 (권역 태그 포함)
6. 각주: policy_snapshot 기반 고지 + "결과는 안내용이며 최종 적격 여부는 프로그램별 CIP 코드 확인 필요"
7. 결과 링크 복사(호주판과 동일: 답변 상태를 URL 파라미터로 인코딩, 접속 시 동일 결과 복원)

## 8. 기술 요구사항

- 정적 사이트 유지(GitHub Pages), 기존 파일 구조·빌드 방식 그대로
- 결과 공유: 기존 호주판 방식 분석 후 동일 구현(URL 쿼리/해시 파라미터)
- 모바일 우선 확인(카카오톡 인앱 브라우저 포함)
- 데이터 로드 실패 시 진단 버튼 비활성 + "잠시 후 다시 시도" 문구
- SEO/OG: title·description·og 태그를 캐나다 컬리지 문구로 교체 (예: "캐나다 컬리지 유학, 전공이 취업허가를 결정합니다 | [브랜드]")

## 9. 작업 순서

- Phase 1: 기존 코드 분석 리포트 (데이터 위치, 진단 상태 관리, 공유 링크 방식 파악) — 코드 수정 없음
- Phase 2: JSON 연결·컬리지 카드 데이터 치환 (호주 대학 → 캐나다 컬리지)
- Phase 3: 진단 질문·추천 로직·PGWP 분기 구현
- Phase 4: 전체 카피 교체(4절 표 + 11절 카피 라이브러리), 정책 고지 섹션 추가
- Phase 5: 로컬 점검 체크리스트 수행 → 커밋 → Pages 확인

## 10. 금지 사항

- CSS·레이아웃·색상 변경, 새 디자인 컴포넌트 추가
- 학비 숫자 임의 기재 (null → "상담 시 안내")
- "영주권 보장"류 보장성 표현, 특정 이민 카테고리 초청 확정 표현
- PGWP 적격 여부를 JSON 밖 정보로 판단
- 퀘벡 컬리지 추가 (푸터에 "퀘벡(CEGEP)은 별도 상담" 한 줄만)

## 11. 카피 라이브러리

- 히어로 훅 후보: ① "같은 컬리지를 나와도, 전공 하나로 취업허가가 갈립니다." ② "졸업 후 3년, 캐나다에서 일하는 전공은 따로 있습니다." ③ "2026년 캐나다 유학, 이제 '어디'보다 '무엇을'이 먼저입니다."
- PGWP 배지 라벨: "PGWP 가능" / "조건부 확인" / "PGWP 불가(비학위)"
- 공통 고지: "본 안내는 2026년 IRCC 발표 기준이며 정책은 변동될 수 있습니다. 프로그램별 최종 적격 여부(CIP 코드)와 최신 학비는 상담 시 확인해 드립니다."
- 편입 트랙 강조: "편입·학위 과정은 전공 제한 없이 취업허가가 가능합니다."
- CTA: "내 전공, 취업허가 되는지 3분 만에 확인" / "1:1 무료 상담 신청"
